# 微信点餐小程序AI客服技术文档

## 一、 项目概述与目标

本项目旨在为微信点餐小程序构建一个集成于Spring Boot后端的高效、智能的AI客服系统。该系统将利用大语言模型（LLM）的能力，结合企业私有知识库（如菜单、优惠活动、配送政策等），实现7x24小时自动响应用户咨询，解答关于点餐流程、菜品信息、订单状态、配送规则等常见问题，从而提升用户体验、降低人工客服成本。核心目标是打造一个“智能预处理+人工深度服务”的混合模式客服体系。

## 二、 系统架构设计

### 2.1 整体技术栈

- **后端框架**：Spring Boot 3.2.0，提供快速开发能力和丰富的生态支持。
- **AI集成框架**：Spring AI 1.0.0-M4。这是一个与OpenAI兼容的AI集成框架，提供了统一的接口来调用不同的AI模型（实用魔搭社区调用api的形式），极大简化了AI功能的集成过程。
- **数据持久层**：MyBatis 3.0.3 + MySQL/H2数据库，用于存储用户对话记录、订单信息等结构化数据。
- **向量数据库**：Milvus或Chroma，用于存储知识库文档（如菜品介绍PDF、运营手册）的向量化索引，实现基于语义的智能检索（RAG）。
- **缓存**：Redis，用于缓存高频问答对、会话状态以及Access Token，提升系统响应速度。
- **前端/小程序**：微信小程序原生开发，通过API与后端Spring Boot服务交互。

### 2.2 微服务分层架构

系统建议采用清晰的分层架构，便于维护和扩展：

1. **API网关层**：使用Spring Cloud Gateway处理请求路由、限流与鉴权。

2. 业务服务层

   ： 

   - **用户/订单服务**：处理小程序核心业务逻辑。
   - **AI客服服务**：本项目的核心，专门处理所有智能对话相关请求。
   - **知识库管理服务**：负责文档的上传、向量化与索引管理。

3. **AI处理层**：以Spring AI为核心，集成DeepSeek等大模型，实现意图识别、知识检索与对话生成。

4. **数据存储层**：MySQL存储业务数据，向量数据库存储知识向量，Redis作为缓存。

## 三、 AI客服核心模块实现

### 3.1 知识库构建与管理

首先，需要将点餐小程序相关的非结构化知识（如菜品详情文档、促销活动规则、常见问题解答FAQ、配送范围说明等）转化为AI可理解的形式。

1. **文档加载与解析**：使用`PDFLoader`、`TextLoader`等工具加载各类文档。
2. **文本分割**：将长文档切分为语义连贯的片段（如段落）。
3. **向量化**：通过嵌入模型（如`all-MiniLM-L6-v2`或大模型提供的嵌入接口）将文本片段转换为向量（Embedding）。
4. **索引存储**：将向量及其对应的原文片段（元数据包含标题、来源等）存入向量数据库（如Milvus）。

### 3.2 智能问答流程（RAG）

当用户在小程序端提问时，后端AI客服服务按以下流程处理：

1. **接收用户问题**：小程序前端通过HTTP API将用户问题`question`和用户ID `userId`发送至后端。
2. **意图识别（可选）**：可先用规则或轻量级模型判断用户意图（如“查询订单”、“咨询菜品”、“投诉建议”），为后续路由提供依据。
3. **向量检索**：将用户问题向量化，在向量数据库中搜索最相似的3-5个知识片段（Top-K相似性搜索）。
4. **提示工程与答案生成**：将用户问题、检索到的相关知识片段、以及历史对话上下文（如使用`ConversationBufferMemory`）组合成一个结构化的提示（Prompt），提交给大语言模型，指令其生成专业、友好的回答。

### 3.3 核心Prompt设计示例

一个有效的Prompt是AI客服回复质量的关键。以下是一个为点餐小程序场景设计的详细Prompt模板，它融合了角色设定、知识背景、输出格式和风格要求：

```
你是一名专业的微信点餐小程序的智能客服助手，名字叫“小餐”。你的核心职责是准确、友好、高效地解答用户关于点餐、订单、配送、菜品和优惠活动的所有问题。

请严格根据以下提供的“权威知识库信息”来组织你的回答。如果知识库中有明确答案，请基于此回答；如果知识库信息不足或未覆盖，你可以运用通用知识进行补充，但必须注明“根据一般情况”。

【权威知识库信息】
{context}

【当前对话历史】
{chat_history}

用户最新提问：{question}

请你按照以下要求生成回复：
1.  **语气与风格**：保持热情、耐心、简洁。使用口语化表达，避免复杂术语。
2.  **结构化**：如果问题涉及多个方面（如菜品口味和价格），请分点说明。
3.  **准确性**：关于菜品配料、价格、促销规则、配送时间和费用等具体信息，必须与知识库完全一致。
4.  **行动引导**：在回答末尾，根据问题自然引导用户下一步操作，例如：“您可以直接在‘菜单’页找到这道菜并下单”，或“如需查看订单详情，请点击‘我的-订单’”。
5.  **免责与转接**：如果用户问题涉及严重投诉、食品安全或无法解决的复杂纠纷，请回复：“您的问题我已记录，为了给您更专业的协助，我将为您转接人工客服，请稍候。”
6.  **格式**：直接输出纯文本，不要使用Markdown、项目符号等任何格式符号。

现在，请开始你的回答：
```

**说明**：

- `{context}`：将由后端从向量数据库检索得到的最相关的知识片段自动填充。
- `{chat_history}`：由对话记忆管理模块提供的近期多轮对话上下文。
- `{question}`：用户当前提出的问题。

### 3.4 后端服务关键代码示例

基于Spring Boot和Spring AI，核心服务层代码示例如下：

**1. 应用配置 (application.yml)**：

```
spring:
  ai:
    openai:
      api-key: ${DEEPSEEK_API_KEY} # 或 wenxin.api-key, tongyi.api-key 等
      base-url: https://api.deepseek.com # DeepSeek API端点
      chat:
        options:
          model: deepseek-chat
          temperature: 0.7 # 控制创造性

# 向量数据库配置 (以Milvus为例)
vectorstore:
  milvus:
    host: localhost
    port: 19530
    collection-name: food_delivery_kb
```

**2. AI客服服务类 (AICustomerService.java)**：

```
@Service
public class AICustomerService {
    @Autowired
    private VectorStore vectorStore; // Spring AI 封装的向量存储接口
    @Autowired
    private ChatClient chatClient; // Spring AI 封装的对话客户端
    @Autowired
    private ConversationBufferMemory memory; // 对话记忆

    public ChatResponse handleUserQuery(String userId, String userQuestion) {
        // 1. 检索相关知识
        List<Document> relevantDocs = vectorStore.similaritySearch(userQuestion);
        String context = relevantDocs.stream()
                                     .map(Doc::getContent)
                                     .collect(Collectors.joining("\n---\n"));

        // 2. 获取历史对话
        String chatHistory = memory.get(userId);

        // 3. 构建完整Prompt
        String systemPrompt = "你是一名专业的微信点餐小程序的智能客服助手..."; // 即上文所述的完整Prompt模板
        String fullPrompt = systemPrompt.replace("{context}", context)
                                        .replace("{chat_history}", chatHistory)
                                        .replace("{question}", userQuestion);

        // 4. 调用大模型
        ChatResponse aiResponse = chatClient.call(fullPrompt);

        // 5. 更新对话记忆
        memory.add(userId, "user", userQuestion);
        memory.add(userId, "assistant", aiResponse.getReply());

        // 6. 返回结果
        return new ChatResponse(aiResponse.getReply(), relevantDocs.stream().map(Doc::getSource).toList());
    }
}
```

**3. REST API控制器 (AIChatController.java)**：

```
@RestController
@RequestMapping("/api/ai-chat")
public class AIChatController {
    @Autowired
    private AICustomerService aiCustomerService;

    @PostMapping("/ask")
    public ResponseEntity<Map<String, Object>> ask(@RequestBody ChatRequest request) {
        ChatResponse response = aiCustomerService.handleUserQuery(request.getUserId(), request.getQuestion());
        Map<String, Object> result = new HashMap<>();
        result.put("reply", response.getReply());
        result.put("sources", response.getSourceDocuments()); // 返回答案溯源
        return ResponseEntity.ok(result);
    }
}
```

## 四、 微信小程序端集成

小程序端主要实现与用户的聊天界面和与后端API的通信。

1. **页面设计**：创建聊天页面，包含消息列表、输入框和发送按钮。
2. **数据交互**：使用`wx.request`或`wx.cloud.callFunction`（如果使用云开发）调用后端Spring Boot提供的 `/api/ai-chat/ask` 接口。
3. **消息展示**：将用户问题和AI回复实时展示在聊天界面中。

## 五、 高级功能与优化建议

1. **人工客服无缝转接**：当AI识别到用户情绪激烈或问题超出处理范围时，可将会话状态和上下文通过Redis同步给在线人工客服坐席，实现平滑接管。

2. 性能优化

   ： 

   - **缓存**：对“营业时间”、“起送价”等高频静态问题答案进行Redis缓存。
   - **异步处理**：对于耗时的向量检索和模型调用，可采用响应式编程（如WebFlux）提升并发吞吐量。

3. 监控与迭代

   ： 

   - 监控关键指标：消息处理延迟、会话接通率、AI问题解决率。
   - 收集用户对AI回复的反馈（如“有用/无用”按钮），用于持续优化知识库和Prompt。

## 六、 部署与运维

1. **本地与服务器部署**：采用 Java 进程方式部署 Spring Boot 应用，MySQL、Redis、Nacos 与向量数据库按独立服务安装和维护，使用启动脚本、系统服务或 CI/CD 流程统一管理。
2. **持续集成/持续部署**：通过GitLab CI/CD或Jenkins实现自动化测试和部署。
3. **知识库更新**：建立流程，当菜单更新或活动规则变更时，重新触发相关文档的向量化与索引更新，实现知识库的零代码维护。

通过以上方案，你可以构建一个功能完整、响应智能、易于维护的微信点餐小程序AI客服系统，显著提升服务自动化水平和用户体验。
