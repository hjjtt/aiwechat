# 任氏外卖微信小程序 - 项目总结文档

## 一、项目概述

**项目名称**：任氏外卖 AI 智能点餐系统

**项目定位**：基于微信小程序的 AI 驱动的外卖点餐系统，采用 Spring Boot 3.2.0 后端 + 微信小程序前端的架构。系统集成 AI 智能客服（ModelScope Qwen3-1.7B 模型），支持菜单浏览、购物车、订单管理、智能客服等功能。

**核心理念**：打造"智能预处理+人工深度服务"的混合模式客服体系，实现 7x24 小时自动响应用户咨询。

## 二、技术架构

### 2.1 整体技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 后端框架 | Spring Boot 3.2.0 | 提供快速开发能力和丰富的生态支持 |
| AI 集成 | Spring AI 1.0.0-M4 | 与 OpenAI 兼容的 AI 集成框架 |
| AI 模型 | ModelScope Qwen3-1.7B | 实用魔搭社区调用 API |
| 数据持久层 | MyBatis 3.0.3 + MySQL | 存储用户对话记录、订单信息等 |
| 向量数据库 | InMemoryVectorStore | 知识库文档向量化存储（生产环境可选 Milvus/Chroma） |
| 缓存 | 内存缓存 | Session 管理（生产环境可选 Redis） |
| 前端/小程序 | 微信小程序原生开发 | 通过 API 与后端 Spring Boot 服务交互 |
| 管理后台 | Vue 3 + Vite | 商家管理系统 |

## 三、核心功能模块

### 3.1 用户认证模块

**功能描述**：实现微信小程序的登录认证功能，支持微信 OAuth 登录和 Mock 登录降级方案。

**核心实现**：
- 微信登录：使用 `wx.login()` 获取 code，后端换取 openid 和 session_key
- Token 管理：生成 72 小时有效期的登录 Token
- 状态检查：每次进入小程序自动检查登录状态

**关键代码位置**：
- `WechatController.java` - 登录/登出/获取用户信息接口
- `WechatLoginService.java` - 微信登录逻辑
- `TokenServiceImpl.java` - Token 生成与验证

### 3.2 菜单管理模块

**功能描述**：提供菜品的浏览、分类筛选、搜索等功能。

**核心功能**：
- 菜品列表：获取所有可售菜品
- 分类筛选：按类别（热菜、素菜、主食、汤类、海鲜）筛选
- 关键词搜索：支持模糊搜索菜品
- 库存状态：显示菜品是否可售

**示例数据**：
| 菜品名称 | 价格 | 分类 | 销量 |
|---------|------|------|------|
| 招牌红烧肉 | ¥38 | 热菜 | 156 |
| 清蒸鲈鱼 | ¥58 | 海鲜 | 89 |
| 宫保鸡丁 | ¥32 | 热菜 | 312 |
| 蛋炒饭 | ¥18 | 主食 | 567 |
| 紫菜蛋花汤 | ¥12 | 汤类 | 423 |

### 3.3 购物车模块

**功能描述**：支持添加/减少商品、数量计算，购物车数据本地存储。

**核心功能**：
- 加入购物车：选择菜品和数量
- 数量调整：支持增加/减少数量
- 价格计算：自动计算小计和总价
- 本地持久化：使用 `wx.setStorageSync` 本地存储

### 3.4 订单管理模块

**功能描述**：完整的订单创建、状态管理、取消订单功能。

**订单状态流转**：
```
pending (待确认) → confirmed (已确认) → preparing (准备中)
       ↓                                    ↓
   cancelled (已取消)              delivering (配送中)
                                          ↓
                                   completed (已完成)
```

**核心功能**：
- 创建订单：选择菜品、填写配送信息
- 订单列表：查看用户所有订单
- 状态筛选：按状态筛选订单
- 取消订单：用户主动取消
- 订单详情：查看订单完整信息

### 3.5 AI 智能客服模块

**功能描述**：基于大语言模型的智能客服系统，结合企业私有知识库实现智能问答。

**AI 问答流程 (RAG)**：
```
用户提问 → 向量化问题 → 向量检索 → 知识库相似匹配
     → 组合 Prompt → 调用 LLM → 生成回答 → 返回结果
```

**技术实现**：
- AI 模型：ModelScope Qwen3-1.7B
- 向量检索：InMemoryVectorStore 内存向量库
- Prompt 工程：融合角色设定、知识背景、对话历史
- 对话记忆：支持多轮上下文

**核心 Prompt 设计**：
```
你是一名专业的微信点餐小程序的智能客服助手，名字叫"小餐"。
你的核心职责是准确、友好、高效地解答用户关于点餐、订单、配送、菜品和优惠活动的所有问题。

请严格根据以下提供的"权威知识库信息"来组织你的回答...
```

**关键服务**：
- `AICustomerService.java` - AI 客服核心服务
- `KnowledgeBaseService.java` - 知识库服务
- `ConversationCacheServiceImpl.java` - 对话上下文缓存

### 3.6 知识库模块

**功能描述**：构建企业私有知识库，为 AI 客服提供准确的业务知识支持。

**知识库内容**：
- 菜单菜品信息（名称、价格、口味、配料）
- 优惠活动规则
- 配送政策说明
- 常见问题解答

**功能实现**：
- 文档上传：支持文本和文件上传
- 向量化存储：文本转换为向量存储
- 相似度检索：基于向量相似度搜索

### 3.7 地址管理模块

**功能描述**：管理用户的收货地址，支持新增、编辑、删除、默认地址设置。

**数据库表**：`user_addresses`
- 联系人信息：姓名、电话
- 地址信息：省市区、详细地址
- 坐标信息：经纬度（用于配送范围判断）
- 标签管理：家、公司、其他

### 3.8 收藏模块

**功能描述**：用户可以收藏喜欢的菜品，方便快速下单。

**核心功能**：
- 添加/取消收藏
- 查看收藏列表
- 切换收藏状态

## 四、数据库设计

### 4.1 核心数据表

#### 4.1.1 用户表 (users)

```sql
CREATE TABLE users (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    open_id VARCHAR(64) NOT NULL UNIQUE,    -- 微信 openid
    nickname VARCHAR(64),                   -- 昵称
    avatar_url VARCHAR(500),                -- 头像URL
    role VARCHAR(20) DEFAULT 'customer',   -- 角色: customer/merchant/admin
    status VARCHAR(20) DEFAULT 'active',   -- 状态
    created_at DATETIME,                    -- 创建时间
    last_login_at DATETIME                  -- 最后登录时间
);
```

#### 4.1.2 会话表 (user_sessions)

```sql
CREATE TABLE user_sessions (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,               -- 用户ID
    token VARCHAR(64) NOT NULL UNIQUE,      -- 登录Token
    expires_at DATETIME NOT NULL,           -- 过期时间
    created_at DATETIME,                    -- 创建时间
    last_active_at DATETIME                 -- 最后活跃时间
);
```

#### 4.1.3 订单表 (orders)

```sql
CREATE TABLE orders (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    order_number VARCHAR(32) NOT NULL UNIQUE,  -- 订单号
    user_id BIGINT NOT NULL,                     -- 用户ID
    status VARCHAR(20) DEFAULT 'pending',        -- 订单状态
    total_amount DECIMAL(10,2),                  -- 订单总金额
    contact_name VARCHAR(50),                    -- 联系人
    contact_phone VARCHAR(20),                   -- 联系电话
    delivery_address VARCHAR(500),               -- 配送地址
    remark VARCHAR(500),                         -- 订单备注
    created_at DATETIME,                         -- 创建时间
    updated_at DATETIME                          -- 更新时间
);
```

#### 4.1.4 订单明细表 (order_items)

```sql
CREATE TABLE order_items (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    order_id BIGINT NOT NULL,            -- 订单ID
    menu_id BIGINT NOT NULL,             -- 菜品ID
    menu_name VARCHAR(100),               -- 菜品名称
    quantity INT,                         -- 数量
    unit_price DECIMAL(10,2),             -- 单价
    subtotal DECIMAL(10,2)                -- 小计
);
```

#### 4.1.5 菜单表 (menus)

```sql
CREATE TABLE menus (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,          -- 菜品名称
    description VARCHAR(500),            -- 菜品描述
    price DECIMAL(10,2) NOT NULL,         -- 价格
    image_url VARCHAR(500),                -- 图片URL
    category VARCHAR(50),                 -- 分类
    is_available BOOLEAN DEFAULT TRUE,    -- 是否可售
    sales_count INT DEFAULT 0            -- 销量
);
```

#### 4.1.6 聊天记录表 (chat_records)

```sql
CREATE TABLE chat_records (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT,                       -- 用户ID
    session_id VARCHAR(64),               -- 会话ID
    question TEXT,                         -- 用户问题
    answer TEXT,                           -- AI 回答
    sources TEXT,                          -- 知识来源
    created_at DATETIME                    -- 创建时间
);
```

#### 4.1.7 用户收藏表 (user_favorites)

```sql
CREATE TABLE user_favorites (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,              -- 用户ID
    menu_id BIGINT NOT NULL,              -- 菜品ID
    created_at DATETIME,
    UNIQUE KEY uk_user_menu (user_id, menu_id)  -- 防止重复收藏
);
```

#### 4.1.8 用户地址表 (user_addresses)

```sql
CREATE TABLE user_addresses (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,              -- 用户ID
    contact_name VARCHAR(50),             -- 联系人
    contact_phone VARCHAR(20),            -- 电话
    province VARCHAR(50),                 -- 省
    city VARCHAR(50),                     -- 市
    district VARCHAR(50),                  -- 区
    address VARCHAR(500),                  -- 地址
    detail_address VARCHAR(500),           -- 详细地址
    latitude DECIMAL(10,7),                -- 纬度
    longitude DECIMAL(10,7),               -- 经度
    is_default BOOLEAN DEFAULT FALSE,      -- 是否默认
    label VARCHAR(20) DEFAULT '家'         -- 标签
);
```

- **前端框架**：Vue 3 + Composition API
- **构建工具**：Vite
- **状态管理**：Pinia
- **路由**：Vue Router

## 十二、总结

任氏外卖微信小程序是一个功能完善的 AI 驱动点餐系统，具备以下特点：

1. **完整的业务闭环**：从登录、浏览菜单、下单、支付到订单管理，全流程支持
2. **AI 智能客服**：基于大语言模型的智能问答，结合私有知识库提供准确服务
3. **良好的架构设计**：分层架构清晰，便于维护和扩展
4. **多端支持**：小程序前端 + Vue 管理后台
5. **容器化部署**：支持 Docker 一键部署

项目采用 Spring Boot 3.2.0 + 微信小程序原生开发，AI 能力通过 ModelScope API 调用 Qwen3-1.7B 模型实现。数据库使用 MySQL，向量知识库目前采用内存存储（可升级为 Milvus/Chroma）。
